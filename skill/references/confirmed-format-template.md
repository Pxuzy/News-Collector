# 简报格式模板（v23 标准 — 2026-07-01 更新）

以下格式是当前最新标准。文件通过 MEDIA: 发送，用 `**粗体标题**` 代替 `#` ATX 标题。

## 核心规则（2026-07-01 确认版）

1. **禁用 `#`/`##`/`###` ATX 标题** — 改用 `**粗体标题**`
2. **禁用 `---` 分隔线**
3. **禁用 Markdown 表格** — 改用 `•` 列表格式（如 `• 平台 — N条 · 状态`）
4. **每条新闻用 1 个 `>` 引用块包裹**（不是多个），块内字段用尾部双空格 `  ` 实现换行
5. `>` 引用块内各字段连续书写，行末加 `  `（两个空格）产生换行，不用空行隔开
6. **论文**例外：每个字段用独立 `>` 块 + 空行隔开
6. **文件只含 `\\n`（LF），不含 `\\r`（CR）**
7. **链接格式**：`[标题](URL)`
8. **所有条目必须有链接、平台排名、热度**

## 🧭 今日主线条目模板

```
🔴 [标题](URL)

> 📍 来源：头条#1 · 热度2074万  
> 📌 事件：发生了什么。包含关键事实和背景（2-3句）  
> 🌊 影响：对谁有影响、影响面（2-3句）  
> 👀 后续：下一步看什么、关键时间节点（1-2句）
```

- 一个 `>` 块，字段间用 `  `（尾部双空格）换行
- 最后一行末尾不需要 `  `

## 🐙 GitHub 条目模板

• 日榜#N [owner/repo](url) ⭐ Stars · 语言

> 📌 项目：一句话概括

```

- 周榜同格式

### GitHub 描述补充规则

- `📌 项目` 字段必须是中文简介，不得直接拼接 GitHub API 返回的英文 `description`。
- 简介应准确描述仓库用途；宽泛关键词不得覆盖更具体的项目类型。无法确认时使用“需结合 README 核验”的保守表述。
- HN 和论文栏目中的标题、字段说明优先使用中文；必要的项目名、模型名和机构名可以保留英文。
- GitHub 栏目按用户偏好只展示 Stars ≥100,000 的项目；不满足门槛时保留中文数据缺口说明，不降低门槛。

### 今日要点事件去重

- `📌 今日要点` 不能只按 URL 或完整标题去重；同一事件的不同角度标题必须使用事件级聚类。
- 事件级去重后再从不同事件补位，避免同一突发事件占据多个要点位置。
- 泛类词只有能唯一绑定到具体事件时才可桥接；无法判断时保留不同条目并在人工检查中标记。

```

- `📌 做什么` 两句合一，句号+空格分隔

## 写入方法

```python
# Windows 唯一正确方式
lines = []
def add(t=""):
    lines.append(t)
# ... 追加内容 ...
with open(path, "w", encoding="utf-8", newline="\\n") as f:
    f.write("\\n".join(lines))
# 清理 \\r
data = open(path, "rb").read().replace(b"\\r", b"")
open(path, "wb").write(data)
```

## 必须避免的错误格式

- ❌ 多个 `>` 块替代单个块（论文除外）
- ❌ `#`/`##`/`###` ATX 标题
- ❌ `---` 分隔线
- ❌ 字段间用 `\\n\\n` 空行（论文除外）
- ❌ 每条新闻没有链接、排名、热度
- ❌ 写入文件含 `\\r`
