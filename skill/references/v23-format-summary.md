# v23 格式规则总结（2026-07-01 确认）

本 session 用户反复修正后确认的最终格式。

## 核心原则

1. **MD 文件模式为唯一默认** — 生成 `.md` 文件通过 `MEDIA:` 发送
2. **禁用 `#`/`##`/`###` ATX 标题** — 用 `**粗体标题**`
3. **禁用 `---` 分隔线**
4. **禁用表格** — 用列表 `• xxx`
5. **禁用 GFM 彩色调用块**（`[!NOTE]`/`[!WARNING]`/`[!TIP]`）

## 引用块规则

- **一条新闻一个 `>` 块**，内部用 `  `（尾部双空格）换行
- **论文例外**：每个字段独立 `>` 块 + 空行隔开
- 引用块最后一行末尾不需要 `  `

### 正确
```
> 📍 来源：头条#1  · 热度2074万  
> 📌 事件：主要内容（2-3句）  
> 🌊 影响：影响分析（2-3句）  
> 👀 后续：下一步看点（1-2句）  
> 💡 建议：可操作建议（1-2句）  
> ✅ 可信度：多源验证
```

### 错误（多个独立 `>` 块）
```
> 📌 事件：xxx

> 🌊 影响：xxx

> 👀 后续：xxx
```

## 字段顺序（固定）

`📍来源 → 📌事件 → 🌊影响 → 👀后续 → 💡建议 → ✅可信度`

- 主线：`📍📌🌊👀`（4字段）
- 国内热点：`📍📌🌊👀💡✅`（全6字段）
- AI·前沿：`📍📌🌊👀✅`（5字段）
- 社媒：`> 💡 解读：`（单字段）
- 论文：`📌 做什么 → 💡 价值 → 💡 工程价值 → 代码`（独立 `>` 块）

## 彩色圆点语义

| 圆点 | 含义 | 适用场景 |
|------|------|---------|
| 🔴 | 最重要/紧急/突破性 | 主线、重大新闻 |
| 🟠 | 次重要/关注 | 普通国内、商业、科技 |
| 🟢 | 正能量/利好 | 暖心故事、市场利好 |
| 🔵 | 经济/金融 | 汇率、股市、通胀 |
| 🟣 | 科技文化/争议 | AI 动态、文化争议 |

## 社媒板块规则

- 条目间**无空行**，紧凑排列
- `1. [标题](url) · 热度\n> 💡 解读`
- 所有平台（抖音/微博/知乎/百度）必须全出

## 写入方法

```python
with open(path, "w", encoding="utf-8", newline="\n") as f:
    f.write("\n".join(lines))
# 清理 \r
data = open(path, "rb").read().replace(b"\r", b"")
open(path, "wb").write(data)
```

**禁止**：`write_file` 工具、heredoc、f-string 嵌套中文引号
