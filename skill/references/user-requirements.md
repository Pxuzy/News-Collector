# 用户需求汇总

格式规则类需求记入 `references/v23-format-summary.md`，本文件仅记录工作流偏好。

## 工作流偏好

1. **默认输出 MD 文件**（`MEDIA:` 发送），不直接发 Telegram 消息文字
2. **生成前数据预检**：检查各板块数据完备性，不足的提前标记跳过
3. **一次性交付**：完整简报一次性生成，不要中途问「继续吗/确认一下」
4. **不要编造**：缺热度写「热度未获取」，缺链接写「链接未获取」，缺排名写「排名未获取」
5. **字段顺序固定**：📍来源 → 📌事件 → 🌊影响 → 👀后续 → 💡建议 → ✅可信度
6. **报告数据缺口**：如实列出哪些数据没拿到、为什么

## 归训（用户曾纠正的事项）

- 不要用 `write_file` 工具写简报文件（Windows 会加 `\r`）
- 不要用 heredoc 或 f-string 嵌套中文引号
- 不要同时输出多个 `>` 引用块（论文除外）
- 社媒条目之间不要有空行
- 不要用 `[!NOTE]`/`[!WARNING]` 等 GFM 彩色块
