# Cron Prompt 精简模板（2026-07-02）

## 任务：生成并发送新闻简报 MD 文件（3次/日：07:00/13:00/19:00）

### 数据获取
1. 先 `python scripts/nb_query.py --status` 检查新鲜度
2. 数据超 2h → 等采集器（约120s）或用 store.query() 直查
3. 循环取 49 源数据（每源 limit=15）

### 生成简报（v23格式，禁止 #/表格/代码块，用 **粗体** + > 引用块 + 彩色圆点）

按顺序生成以下板块（每条必须有链接、排名、热度、💡 解读）：
1. 🧭 今日主线（3-5条，聚合热度）
2. 🏮 国内7子板块：时政/财经/科技/汽车/娱乐/体育/社会
   - 每条完整字段：📍/📌/🌊/👀/💡/✅
3. 🌏 国外热点（国际+财经）
4. 🔥 社媒热榜（跨平台分析 + 抖音/微博/知乎/百度独立输出）
5. 🤖 AI·前沿
6. 🐙 GitHub 日榜TOP5+周榜TOP10（含🧩解读）
7. 🐱 HackerNews TOP10（Points+💡解读）
8. 📜 论文·学术（8篇，📌两句+💡价值+💡工程价值+代码）
9. 📊 平台统计 + 🤖 全景判断 + 👀 继续跟踪（带💡）+ ⚠️ 风险与机会 + 🧩 数据缺口

### 格式要点
- 主线/国内：同一 > 块，尾部双空格换行
- 论文：各字段独立 > 块 + 空行
- GitHub：`• 日榜#N [repo](url) ⭐ Stars · 语言` + `> 📌/🧩/🔥/👀`
- HN：`• HN#N [title](url) Points：N · Comments：N · 类型` + `> 💡`
- 继续跟踪每条加 `> 💡 跟踪：`
- 风险：至少2风险+2机会

### 交付
1. 写 Python 脚本 → 执行产出 MD 文件 → 清理 \r
2. 保存到 `cron/output/news-YYYY-MM-DD.md`
3. 校验：`python scripts/validate_briefing.py "E:/path/to/file.md"`
4. 一句话回复 + `MEDIA:E:/path/to/file.md`
5. **严禁在聊天框贴长篇正文**
6. 不编造数据，缺口如实标注，不要问问题
