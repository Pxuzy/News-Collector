# 简报定时优化指南（2026-07-02）

## 数据驱动的定时选择

简报推送时间应根据**新闻数据积累曲线**而非固定习惯决定。通过分析 DB 中按小时新增的新闻量来找到最佳推送窗口。

### 查询方法

```sql
-- 查最近7天按小时的新增新闻量
SELECT substr(first_seen, 12, 2) as hour,
       count(*) as items
FROM news_items
WHERE first_seen > date('now', '-7 days')
GROUP BY hour
ORDER BY hour;
```

### 本用户的积累曲线（2026-07-02 实测）

| 小时 | 新增量 | 特征 |
|------|--------|------|
| 08:00 | 548 | 🏆 早间峰值 |
| 13:00 | 531 | 🏆 午间峰值 |
| 18:00-20:00 | 481/417/412 | 🏆 晚间峰值 |

### 最佳推送时间推导

原则：采集在推送前 5 分钟执行，推送应在新闻积累峰值**之后**（内容已产出）且**在用户空闲时**送达。

| 推送 | 采集时间 | 推送时间 | 覆盖的峰值 | 用户场景 |
|------|---------|---------|-----------|---------|
| 早报 | 06:55 | 07:00 | 08:00 峰值的尾声 + 前夜新间 | 早餐/通勤 |
| 午报 | 12:55 | 13:00 | 13:00 午间峰值 | 午休 |
| 晚报 | 18:55 | 19:00 | 18:00-20:00 晚间三个峰值 | 晚餐/下班 |

### Cron 表达式（2026-07-02 更新 — 分流采集 + 回顾晚报）

```
# 全量采集（50 源，早晚各一次）
news-toolkit-collector:      55 6,18 * * *
# 核心采集（~30 源，午间快速刷新）
news-toolkit-core-collector: 55 12 * * *
# 标准简报（07:00 / 13:00）
enhanced-news-briefing:      0 7,13 * * *
# 晚间回顾（19:00 — 今日回顾模式）
evening-news-recap:          0 19 * * *
```

### 采集分流策略

- **06:55 / 18:55** → `run_collector_silent.py`（全量 50 源，~120s）
- **12:55** → `run_collector_core.py`（核心 ~30 源，~60s，午间快速刷新）
- 采集后自动运行 `fix_data_gaps.py` 补齐 HN 和 GitHub 字段

## Cron Agent Prompt 精简原则

`enhanced-news-briefing` 的 cron prompt 必须是**可执行指令清单**而非长文说明：

### 正确结构（约 2000 字符）

```
## 任务：xxx

### 数据获取
1. 命令 A
2. 命令 B

### 生成简报（按顺序）
1. 🧭 主线（3-5条）
2. 🏮 国内7子板块
   - 字段要求：📍/📌/🌊/👀/💡/✅
...

### 格式规则
- 禁止 #/表格/代码块
- 论文用独立 > 块

### 交付
1. 写入 → 校验 → MEDIA
2. 一句话回复
```

### 错误结构（过长，不建议）

```
（整份 SKILL.md 原文，数万字符）
```
